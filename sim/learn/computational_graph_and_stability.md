# 计算图、雅可比矩阵与控制系统稳定性

> 本文解释一个深层联系：**决定控制系统是否稳定的数学结构（雅可比矩阵的特征值），恰好也决定了反向传播时梯度是否爆炸或消失**。这不是巧合——它们是同一枚硬币的正反面。
>
> 建议先阅读 `backpropagation_through_control_loop.md`，本文在其基础上从控制论视角深入。

---

## 1. 同一个矩阵，两种解读

在我们的控制循环中，每一步都有一个状态转移：

```
state_{t+1} = f(state_t, δ_t, a_t)
```

对这个函数求**偏导数**（state 对 state），得到一个矩阵——**雅可比矩阵**（Jacobian）：

```
J_t = ∂state_{t+1} / ∂state_t
```

这个矩阵是 4×4 的（因为 state = [x, y, yaw, v] 有 4 个分量），它描述了"state 的微小变化如何传播到下一步"。

**关键洞察**：这同一个矩阵，控制工程师和机器学习工程师各看到了不同的东西：

| 视角 | 问的问题 | 数学表达 | 读法 |
|------|----------|----------|------|
| **控制论（正向）** | 如果初始状态有微小偏差 Δ，N 步后偏差变多大？ | Δstate_N = (∏ J_t) · Δstate_0 | 连乘放大 → 不稳定 |
| **机器学习（反向）** | loss 对初始状态的梯度有多大？ | ∂loss/∂state_0 = ∂loss/∂state_N · (∏ J_t) | 连乘放大 → 梯度爆炸 |

两者都归结为**同一个雅可比矩阵的 N 次连乘**。连乘的行为由 J 的**特征值** λ 决定：

- |λ| > 1：连乘指数增长 → **正向：不稳定** / **反向：梯度爆炸**
- |λ| < 1：连乘指数衰减 → **正向：稳定** / **反向：梯度消失**
- |λ| = 1：连乘不增不减 → **正向：边际稳定** / **反向：梯度刚好**

## 2. 雅可比矩阵：它到底长什么样

### 2.1 从自行车模型推导

我们项目的车辆模型（`sim/model/vehicle.py`）：

```python
x_{t+1}   = x_t   + v_t · cos(yaw_t) · dt
y_{t+1}   = y_t   + v_t · sin(yaw_t) · dt
yaw_{t+1} = yaw_t + v_t · tan(δ_t) / L · dt
v_{t+1}   = v_t   + a_t · dt
```

"雅可比矩阵"就是对每个输出分量，分别对每个输入分量求偏导，排成矩阵：

```
J_t = ∂state_{t+1} / ∂state_t

     ∂x'    ∂x'      ∂x'     ∂x'
   = ───    ───      ───     ───
     ∂x     ∂y      ∂yaw     ∂v

     ∂y'    ∂y'      ∂y'     ∂y'
     ───    ───      ───     ───
     ∂x     ∂y      ∂yaw     ∂v

     ∂yaw'  ∂yaw'   ∂yaw'   ∂yaw'
     ───    ───      ───     ───
     ∂x     ∂y      ∂yaw     ∂v

     ∂v'    ∂v'      ∂v'     ∂v'
     ───    ───      ───     ───
     ∂x     ∂y      ∂yaw     ∂v
```

逐项计算（别被公式吓到，每一项都只是简单的求导）：

```
       ┌                                               ┐
       │  1      0     -v·sin(yaw)·dt    cos(yaw)·dt   │
       │                                               │
       │  0      1      v·cos(yaw)·dt    sin(yaw)·dt   │
J_t =  │                                               │
       │  0      0          1            tan(δ)/L·dt   │
       │                                               │
       │  0      0          0                1         │
       └                                               ┘
```

### 2.2 怎么读这个矩阵

**对角线全是 1**：每个分量"记住"自己的上一步值。这意味着 J ≈ I（单位矩阵），状态是"积累型"的。

**非对角项**是耦合：
- `∂x'/∂yaw = -v·sin(yaw)·dt`：yaw 的变化会影响下一步 x 的变化方向
- `∂x'/∂v = cos(yaw)·dt`：速度影响位移大小
- `∂yaw'/∂v = tan(δ)/L·dt`：速度影响转弯速率

**下三角的 0**：v 不受 x, y, yaw 影响（纯运动学模型中，速度只由加速度决定）。

### 2.3 "I + dt·A" 的形式

可以把 J 写成：

```
J = I + dt · A
```

其中 A 是一个小矩阵。因为 dt = 0.02（50Hz），A 的元素再乘 0.02 后都很小。这意味着 **J 的特征值非常接近 1**。

具体来说：J 是上三角矩阵，特征值就是对角线元素 = **全部为 1**。

这就是**边际稳定**：扰动不放大，也不衰减。纯运动学的车辆模型就像在冰面上滑行——你推一下它，它就一直滑下去。

## 3. 开环 vs 闭环：控制器如何改变一切

### 3.1 开环（无控制器）

如果 δ 和 a 是预先设定好的常数（不随 state 变化），那么 J 就是上面推导的样子——特征值全为 1。

**正向含义**：初始位置偏了 1 米，500 步后仍然偏 1 米（不纠正）。

**反向含义**：梯度连乘 1^500 = 1，不爆炸也不消失。

### 3.2 闭环（有控制器）

在我们的系统中，转向角 δ 是控制器的输出，它**依赖于当前状态**：

```
δ_t = π(state_t; θ)    ← 控制器是 state 的函数
```

因此，状态转移变成：

```
state_{t+1} = f(state_t, π(state_t; θ))
```

对 state_t 求全微分时，要用**链式法则**：

```
dstate_{t+1} / dstate_t = ∂f/∂state + ∂f/∂δ · ∂π/∂state
                          ─────────   ──────────────────
                          开环部分       控制器反馈部分
```

**控制器反馈部分**就是关键：一个好的控制器会引入**负反馈**。

### 3.3 什么是负反馈

以最简单的比例控制为例：

```
δ = -K × lateral_error    （误差向左，方向盘向右打——负号是关键）
```

这个负号使得 `∂δ/∂state` 带有"纠错方向"的符号，从而让闭环雅可比的特征值**从 1 减小到 < 1**。

直觉上很好理解：
- 车偏左了 → 控制器把方向盘向右打 → 车回到中间 → 偏差被消除
- 这就是"扰动会衰减"——稳定的定义

用特征值的语言：负反馈把 λ 从 1 拉到了 < 1。反馈增益 K 越大，λ 越小，系统响应越快。

## 4. 核心矛盾：稳定性 vs 可学习性

现在可以揭示最深层的联系了：

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   好的控制器 → |λ| < 1 → 系统稳定 ✓  但梯度消失 ✗           │
│                                                              │
│   差的控制器 → |λ| > 1 → 系统不稳定 ✗  且梯度爆炸 ✗         │
│                                                              │
│   完美的悖论：                                                │
│   你越想让系统稳定（控制目标），间接梯度就越容易消失（学习困难） │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

为什么会这样？因为**同一个雅可比矩阵**既控制了扰动的传播（正向），也控制了梯度的传播（反向）。

**稳定意味着"忘记过去"**：一个稳定的控制系统会主动消除初始偏差——500 步后你几乎看不出初始状态偏了多少。这对控制来说很好，但对学习来说意味着"loss 对很久以前的参数选择几乎没有感觉"——因为那个影响已经被控制器自己消除了。

**不稳定意味着"放大一切"**：一个不稳定的系统会把微小的参数变化放大成巨大的状态偏差。这对学习来说是"强信号"，但对控制来说是灾难——系统会发散。

### 数值体感

假设闭环系统的典型特征值 λ = 0.99（稳定且接近边界）：

| 步数 | λ^N | 含义 |
|------|------|------|
| N = 10 | 0.90 | 10 步后，初始偏差衰减 10%——间接梯度还行 |
| N = 50 | 0.61 | 还有六成信号——勉强可用 |
| N = 100 | 0.37 | 只剩三成七——变弱了 |
| N = 300 | 0.05 | 几乎没了——间接梯度实际消失 |
| N = 500 | 0.007 | 不到百分之一——完全消失 |

反过来，如果 λ = 1.01（微微不稳定）：

| 步数 | λ^N | 含义 |
|------|------|------|
| N = 10 | 1.10 | 轻微放大——没问题 |
| N = 50 | 1.64 | 还好 |
| N = 100 | 2.70 | 开始显著 |
| N = 300 | 19.8 | 大了——梯度开始不受控 |
| N = 500 | 144.8 | 严重爆炸——训练 NaN |

仅仅从 0.99 到 1.01，差了 0.02，但 500 步后行为天差地别（0.007 vs 144.8）。

**这就是指数效应的残酷**：连乘放大（或缩小）的力量随步数指数增长。

## 5. 为什么我们的系统仍然能学习

既然稳定系统的间接梯度会消失，我们是怎么训练成功的？答案在于**两条梯度路径**。

### 5.1 直接路径 vs 间接路径

回顾 `backpropagation_through_control_loop.md` 第 6 节的结论：

```
间接路径（经过状态链，受雅可比连乘影响）：
  θ ──→ δ₀ ──→ state₁ ──→ state₂ ──→ ... ──→ stateₜ ──→ errorₜ ──→ loss
          └──── 经过 t 个雅可比矩阵连乘，可能消失/爆炸 ────┘

直接路径（只经过 1 步，不涉及雅可比连乘）：
  θ ──→ δₜ ──→ errorₜ ──→ loss
          └──── 只有 1 步，始终稳定健康 ────┘
```

**直接路径是我们的救命稻草**：每一步 t，参数 θ 都直接决定了当步的转向角 δₜ（通过查找表、PID 计算等）。这条路径只涉及当步内的计算，**不经过任何雅可比连乘**，因此梯度不会爆炸也不会消失。

换句话说：
- 间接路径告诉你"早期的参数选择如何通过状态传播影响未来"——在稳定系统中这条信息很弱
- 直接路径告诉你"当前的参数值如何影响当前的输出"——这条信息始终清晰

对于我们的控制器调参任务，直接路径提供的信息通常已经足够：**你在 step 200 时 T2 表查出的 prev_time_dist 如何影响 step 200 的转向精度**。

### 5.2 Truncated BPTT 的物理意义

现在可以从控制论视角理解 Truncated BPTT 了：

```python
if step % K == 0:
    car.detach_state()    # 切断状态链
```

**它做的事**：限制间接路径的最大长度为 K 步。

**控制论解读**：只保留 K 步以内的"状态记忆"。对于特征值 λ = 0.99 的系统：
- K = 64：λ^64 = 0.52，间接梯度保留了一半——有用
- K = 128：λ^128 = 0.27——还行
- K = 256：λ^256 = 0.08——大部分已经消失了，截不截差别不大

所以对于稳定系统，**Truncated BPTT 的 K 值不需要太大**——因为超过几十步后间接梯度自然消失，再长也提供不了多少额外信息。

**反过来**，如果系统不太稳定（λ ≈ 1.0 或 > 1），K 就需要小一些来防止梯度爆炸。

### 5.3 梯度裁剪（Gradient Clipping）

在 `sim/optim/train.py` 中：

```python
clip_grad_norm_(params.parameters(), max_norm=10.0)
```

这是对抗梯度爆炸的最后防线。如果某些步骤的雅可比特征值瞬时 > 1（比如高速急转弯时），间接路径的梯度可能在局部爆炸。梯度裁剪把梯度的总范数限制在 10 以内，保证优化器不会收到荒谬的更新。

## 6. 完整的图景

三者的关系可以这样总结：

```
             ┌─────────────────────────────────────────────┐
             │            计 算 图                          │
             │  (PyTorch 自动构建的 DAG，记录所有运算)       │
             │                                             │
             │   θ ──→ 控制器 ──→ δ                         │
             │                    ↓                        │
             │   state_t ──→ vehicle.step ──→ state_{t+1}  │
             │                                    ↓        │
             │                              error → loss   │
             └─────────────┬───────────────────────────────┘
                           │
                   loss.backward()
                  沿着图反向传播
                           │
                           ▼
             ┌─────────────────────────────────────────────┐
             │          雅 可 比 矩 阵                      │
             │   J_t = ∂state_{t+1} / ∂state_t             │
             │                                             │
             │   正向看：状态扰动的传播系数                   │
             │   反向看：梯度的传播系数                       │
             │                                             │
             │   两者是同一个矩阵，但沿相反方向使用           │
             └─────────────┬───────────────────────────────┘
                           │
                    特征值 λ 决定
                           │
               ┌───────────┼───────────┐
               │           │           │
               ▼           ▼           ▼
          |λ| > 1      |λ| ≈ 1     |λ| < 1
             │           │           │
     ┌───────┴──┐   ┌───┴───┐   ┌──┴────────┐
     │ 正向：    │   │边际    │   │ 正向：     │
     │ 系统不稳定│   │稳定    │   │ 系统稳定   │
     │          │   │       │   │           │
     │ 反向：    │   │梯度    │   │ 反向：     │
     │ 梯度爆炸  │   │刚好    │   │ 梯度消失   │
     └──────────┘   └───────┘   └───────────┘
```

## 7. 一句话总结

> **雅可比矩阵是计算图中相邻两步之间的"放大镜"。控制工程师用它看扰动是否发散（稳定性），机器学习用它看梯度是否爆炸（可训练性）。一个稳定的控制系统天然压制间接梯度，但直接路径（参数→当步输出→当步误差）不受影响，这就是可微控制能训练成功的根本原因。**

## 8. 常见困惑 Q&A

### Q1: 雅可比矩阵和普通导数有什么区别？

普通导数是标量对标量的：`dy/dx = 2`。

雅可比矩阵是**向量对向量的导数**。当输入是一个向量 [x, y, yaw, v]，输出也是一个向量 [x', y', yaw', v'] 时，每一对 (输出分量, 输入分量) 都有一个偏导数。把这些偏导数排成矩阵，就是雅可比矩阵。

它的每一行描述"某一个输出如何依赖所有输入"，每一列描述"某一个输入如何影响所有输出"。

### Q2: 特征值是什么？为什么它决定了连乘的行为？

特征值是矩阵的"内在缩放因子"。一个矩阵 J 可能在某些方向拉伸（λ > 1），在另一些方向压缩（λ < 1）。

连乘 N 次时，最大特征值主导行为：如果 λ_max > 1，沿该方向指数增长，N 次后不管其他方向怎样，这个方向的分量会压倒一切。

类比：想象你每天工资涨 1%。看似不多，但 `1.01^365 ≈ 37.8`——一年后涨了 37 倍。这就是连乘的指数效应。

### Q3: 为什么自行车模型的特征值是 1 而不是 > 1 或 < 1？

因为自行车模型是**纯运动学**的——它只描述"速度产生位移、转向角产生转弯"，没有摩擦、没有阻力、没有任何耗散机制。在这种理想化模型中：

- 位移是速度的积分 → 速度偏差不会消除，也不会放大 → λ = 1
- yaw 是角速度的积分 → 同理 → λ = 1

现实中有摩擦和空气阻力（λ < 1），但在运动学近似下被忽略了。控制器提供的负反馈代替了这些自然耗散力。

### Q4: 控制器增益 K 越大越好吗？

不是。K 越大，闭环特征值越小（越远离 1），系统越稳定、响应越快。但 K 过大会导致：

1. **控制输入饱和**：方向盘打到底了还不够
2. **高频振荡**：过度纠正导致左右摇摆
3. **间接梯度消失更快**：λ 太小，`λ^K` 很快趋零，失去长程依赖信息

控制工程的核心就是找到合适的 K——既足够稳定，又不过度。我们的可微控制框架用梯度下降来自动寻找这个平衡。
